Checkbox

Checkboxes provide a single binary (yes/no) input field which can be required. Checkbox groups can be used for multiple choice selections.

Sources:

  • Checkbox Github
  • CheckboxGroup Github

Usage Guide#

Installation#

yarn add @twilio-labs/match formik

Checkbox Usage#

import { Formik } from "formik";
import { Checkbox } from "@twilio-labs/match/components";
const Component = () => (
<Formik
initialValues={{
accept_terms: [],
}}
onSubmit={() => {}}
>
<Checkbox
name="accept_terms"
value="1"
label="Do you accept the terms of service?"
required
/>
</Formik>
);
Note: The value will always be an array, even when working with a single checkbox#

CheckboxGroup Usage#

import { Formik } from "formik";
import { Checkbox, CheckboxGroup } from "@twilio-labs/match/components";
const Component = () => (
<Formik
initialValues={{
animals: [],
}}
onSubmit={() => {}}
>
<CheckboxGroup
label="What animals do you like?"
additional="Please select at least one option"
required
>
<Checkbox name="animals" value="dog" label="Dogs" />
<Checkbox name="animals" value="cat" label="Cats" />
<Checkbox name="animals" value="rabbit" label="Rabbits" />
<Checkbox name="animals" value="frog" label="Frogs" />
</CheckboxGroup>
</Formik>
);

Guidelines#

About Inputs#

Checkboxes can be used independently or within a CheckboxGroup with multiple options. If you intend to have multiple options with only one available selection, use a RadioGroup instead.

Accessibility#

  • Checkboxes and CheckboxGroups should be described with label props.
  • Use helper text to provide extra context when needed.
  • Keep labels and helper text brief and descriptive for users using assistive technology, so they can understand the question clearly.

Composition#

  • Use CheckboxGroup components to group multiple related choices.

  • Checkboxes must have both name and value props.

  • Checkboxes within a checkbox group may have different name props.

Do
Use checkboxes in a checkbox group with at least two or more choices - where a user may select more than one option.
Don't
Don't use a single checkbox within a checkbox group.
Do
List options in a logical order that makes sense to your users such as in order of the most common selections or in numerical order.
Don't
Don't randomize the list options in a way that users will find confusing.
Do
Consider using the multi-select component when list options exceed around 10 choices.
Don't
Don't have a long list of checkboxes that may overwhelm users.

Examples#

We currently serve three different themes: Twilio, SendGrid, and Ahoy.

Individual Checkbox#

Individual checkboxes require name, value, and label props.

<Formik
  initialValues={{
    accept_terms: [],
  }}
  onSubmit={() => {}}
>
  <Checkbox
    name="accept_terms"
    value="1"
    label="Do you accept the terms of service?"
    required
  />
</Formik>

Vertical Checkbox Group#

The default Checkbox Group style is vertically stacked. This style of Checkbox Group works well for longer content.

How would you like to receive notifications?
We will only call during business hours
Messaging rates may apply
Please check your junk folder
There is a small surcharge for unreturned pigeons
<Formik
  initialValues={{
    notifications: ["phone", "sms"],
  }}
  onSubmit={() => {}}
>
  <CheckboxGroup label="How would you like to receive notifications?" required>
    <Checkbox
      name="notifications"
      value="phone"
      label="Phone Call"
      additional="We will only call during business hours"
    />
    <Checkbox
      name="notifications"
      value="sms"
      label="SMS"
      additional="Messaging rates may apply"
    />
    <Checkbox
      name="notifications"
      value="email"
      label="Email"
      additional="Please check your junk folder"
    />
    <Checkbox
      name="notifications"
      value="pigeon"
      label="Carrier Pigeon"
      additional="There is a small surcharge for unreturned pigeons"
    />
  </CheckboxGroup>
</Formik>

Horizontal Checkbox Group#

The horizontal prop can be used to give the CheckboxGroup a horizontal layout. This is best for closely related groups with few options. Be mindful of the width of the group, as wrapping may occur.

What is your preferred meeting day
<Formik
  initialValues={{
    meeting: [],
  }}
  onSubmit={() => {}}
>
  <CheckboxGroup label="What is your preferred meeting day" horizontal>
    <Checkbox name="meeting" value="monday" label="Monday" />
    <Checkbox name="meeting" value="wednesday" label="Wednesday" />
    <Checkbox name="meeting" value="friday" label="Friday" />
  </CheckboxGroup>
</Formik>

Readonly#

The readOnly prop can be used for displaying selections back to the user.

What is your preferred meeting day
<Formik
  initialValues={{
    meeting: ["monday"],
  }}
  onSubmit={() => {}}
>
  <CheckboxGroup label="What is your preferred meeting day" readOnly>
    <Checkbox name="meeting" value="monday" label="Monday" />
    <Checkbox name="meeting" value="wednesday" label="Wednesday" />
    <Checkbox name="meeting" value="friday" label="Friday" />
  </CheckboxGroup>
</Formik>

Disabled#

The disabled prop can be used for disallowing user input. Note: Disabled fields are not submitted with the form data.

What is your preferred meeting day
<Formik
  initialValues={{
    meeting: ["monday"],
  }}
  onSubmit={() => {}}
>
  <CheckboxGroup label="What is your preferred meeting day" disabled>
    <Checkbox name="meeting" value="monday" label="Monday" />
    <Checkbox name="meeting" value="wednesday" label="Wednesday" />
    <Checkbox name="meeting" value="friday" label="Friday" />
  </CheckboxGroup>
</Formik>

Required#

An error will be displayed if a choice is not selected.

<Formik
  initialValues={{
    accept_terms: [],
  }}
  validateOnMount
  initialTouched={{ example: true }}
  onSubmit={() => {}}
>
  <Checkbox
    name="accept_terms"
    value="1"
    label="Do you accept the terms of service?"
    required
  />
</Formik>

Validation#

The default validation can be overridden using the validate prop. Validation can be turned off using the noValidate prop.

What are your preferred meeting days?
<Formik
  initialValues={{
    meeting: ["sunday", "monday"],
  }}
  onSubmit={() => {}}
>
  <CheckboxGroup
    label="What are your preferred meeting days?"
    validate={(value) => {
      if (value.length < 2) {
        return "Please select at least two options.";
      }
    }}
  >
    <Checkbox
      name="meeting"
      value="sunday"
      label="Sunday (deselect to see error)"
    />
    <Checkbox name="meeting" value="monday" label="Monday" />
    <Checkbox name="meeting" value="wednesday" label="Wednesday" />
    <Checkbox name="meeting" value="friday" label="Friday" />
  </CheckboxGroup>
</Formik>
